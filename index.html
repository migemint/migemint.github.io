PEP 8 编码规范

布局

代码缩进

使用四空格.

当代码连续多行时，我们应该使用小括号，中括号，大括号或者悬挂缩进对其隐式行链接.当使用悬挂缩进时应当保证：第一行没有代码，接下来的缩进明确的区分自己作为接下来的行.

YES:

# 与开头的分隔符对齐.
foo = long_function_name(var_one, var_two,
                                             var_three, var_four)

# 使用更多的缩进与后续代码区分.
def long_function_name(
            var_one, var_two, var_three,
            var_four):
    print(var_one)

# 增加一级悬挂缩进l.
foo = long_function_name(
    var_one, var_two,
    var_three, var_four)

NO:

# 当纵向没有对齐时第一行不允许有参数.
foo = long_function_name(var_one, var_two,
    var_three, var_four)

# 由于缩进并不鲜明所以需要更多的缩进.
def long_function_name(
    var_one, var_two, var_three,
    var_four):
print(var_one)

四空格规则对于连续多行也是可选的.

Optional:

# Hanging indents *may* be indented to other than 4 spaces.
foo = long_function_name(
    var_one, var_two,
    var_three, var_four)

当if表达式的条件部分太长时就需要跨越多行，使用“if (”构造一个人工的4-space缩进是不提倡的.这会与嵌套在if表达式内的代码构成视觉冲突，因为这部分代码也是四个空格的缩进.、
可用的方式有：

# 没有额外的缩进的情况，会产生视觉冲突.
if (this_is_one_thing and
    that_is_another_thing):
    do_something()

# 添加注释可以产生一些视觉的区分
# 支持语法高亮.
if (this_is_one_thing and
    that_is_another_thing):
    # Since both conditions are true, we can frobnicate.
    do_something()

#在后续的条件行前添加额外的缩进.
if (this_is_one_thing
        and that_is_another_thing):
    do_something()

多行构造器的闭合大/中/小括号或者与上一行的第一个非空字符对齐：

my_list = [
    1, 2, 3,
    4, 5, 6,
    ]
result = some_function_that_takes_arguments(
    'a', 'b', 'c',
    'd', 'e', 'f',
    )

或者与多行构造器第一行的第一个字母对齐：

my_list = [
    1, 2, 3,
    4, 5, 6,
]
result = some_function_that_takes_arguments(
    'a', 'b', 'c',
    'd', 'e', 'f',
)

Tabs或者Spaces

像比较来说，Spaces用于缩进更好.

Tabs只有当剩余的代码已经使用Tabs的情况下才能使用.

Python 3 不允许tabs和spaces混用.

Python 2 混用的情况应该转换为使用spaces

当调用Python命令行使用 -t 选项时，混用会产生warning; 使用 -tt 时， 混用产生错误.

最大行长度

限制所有行不超过79个字符.

对于只有较少限制（docstrings or comments）的较长的块，行长度应该不超过72个字符.

限制需要的编辑器窗口宽度可以同时打开多个文件，并且当使用代码检查工具对比两个版本的代码时可以很方便.

大多数工具中默认的折叠都会破坏代码的视觉结构，从而很难看懂.一些基于web的工具甚至可能不提供动态的行折叠.

一些团队更偏爱较长的行长度.对于代码的维护专门或主要由一个团队负责并能在这个问题达成协议时，行长度从80涨到100也是可以的,注释和文档仍然被折叠到72个字符.

Python标准库很保守需要限制行长度到79个字符（文档/注释到72个字符）.

确定能够恰当的缩进连续行.打断一个二进制操作符是在操作符之后，而不是之前：
class Rectangle(Blob):

def __init__(self, width, height,
             color='black', emphasis=None, highlight=0):
    if (width == 0 and height == 0 and
            color == 'red' and emphasis == 'strong' or
            highlight > 100):
        raise ValueError("sorry, you lose")
    if width == 0 and height == 0 and (color == 'red' or
                                   emphasis is None):
        raise ValueError("I don't think so -- values are %s, %s" %
                        (width, height))
  Blob.__init__(self, width, height,
                        color, emphasis, highlight)

空行

顶级的函数或者类定义使用两个空行分隔.

类内方法使用单空行隔开.

额外的空行可能会被用于相关函数的分组.空行可能可以省去一堆俏皮话.

在函数中使用空白行来指明逻辑上的区分.

Python将control-L换页符看做空白；许多工具将其作为分页符，所以你可以使用他们将文件中的相关部分分成几页.注意，一些编辑器以及基于web的阅读器可能不将control-L视为换页而是显示另外一种雕文(glyph).

源文件编码

在Python中的核心程序一般使用UTF-8（or ASCII in Python 2）.

使用ASCII（in Python 2）或者UTF-8（in Python 3）不使用编码声明.

在标准库中, 非默认编码仅仅应该用于测试目的或者文档需要提到包含非ASCII编码字符的作者姓名；否则使用\x, \u, \U or \N 来替代在字符文字中使用非ASCII数据.

对于Python 3.0以及后续版本， 接下来的条目将是规定：所有Python标准库MUST中的标识符使用ASCII-only的标识符，并且应该使用英文单词.另外string字符和注释也必须使用ASCII字符.仅有的例外与上面提到的例外一致.

导入

导入应该被分割为多行

#YES
import os
import sys

#NO   
 import sys, os

这种方式是允许的:

from subprocess import Popen, PIPE

导入永远放在文件的开头，在任意模块注释和文档之后，在模块全局变量和常量之前.

导入应该被分为如下组：

  1. 标准库导入
  2. 相关第三方导入
  3. 本地应用/库导入

应该用空行将每组导入隔开.

将任意相关的 _al _ 放在导入后.

推荐使用绝对导入，因为这样往往更具有可读性并且在导入系统错误配置时会有更好的行为（或者至少是更好的错误信息）.

import mypkg.sibling
from mypkg import sibling
from mypkg.sibling import example

然而显式相对导入也是可以接收的，尤其是使用绝对引入在处理复杂报结构时可能会过于冗长：

from . import sibling
from .sibling import sample

标准库代码应该避免复杂的包结构并且总是使用绝对导入.

隐式相对导入不应该被使用并且在Python 3中已经被移除了.

通配符导入（from import *）应该避免，因为他们会使得命名空间中的名字混淆，迷惑读者以及许多自动工具.这里有一个使用通配符导入的特例,当从新发布一个内部接口作为公共API的一部分时.

当使用这种方式重新发布名字时，下面的指南对于内部或外部借口同样适用.

引号

在Python中，单引号字符串与双引号字符串是相同的.本标准不推荐这样使用.选择一个规则（总是使用单引号或者双引号）并且一以贯之.当字符串包含单引号或者双引号时，使用另一个从而避免在字符串中出现反斜杠.这样会更有可读性.

在表达式中使用空格

小毛病

避免如下情况种多余的空格：

  • 在大括号，中括号和小括号中.

    YES: spam(ham[1], {eggs: 2})
    No: spam( ham[ 1 ], { eggs: 2 } )

  • 在逗号，分号，冒号之前.

    YES: if x == 4: print x, y; x, y = y, x
    NO: if x == 4: print x , y ; x , y = y , x

  • 当分片时，冒号就像一个二值操作符，其两边的数量应该相等.在拓展分片时，分号应该有相同数量的空格.

例外： 当参数被省略时，空格也被省略.

YES:
ham[1:9], ham[1:9:3], ham[:9:3], ham[1::3], ham[1:9:]
ham[lower:upper], ham[lower:upper:], ham[lower::step]
ham[lower+offset : upper+offset]
ham[: upper_fn(x) : step_fn(x)], ham[:: step_fn(x)]
ham[lower + offset : upper + offset]

NO:
ham[lower + offset:upper + offset]
ham[1: 9], ham[1 :9], ham[1:9 :3]
ham[lower : : upper]
ham[ : upper]

  • 在函数调用的参数列表的左括号前不加空格

    YES： spam(1)
    NO : spam (1)

  • 在索引或分片的左中括号前不加空格

    YES： dct[‘key’] = lst[index]
    NO : dct [‘key’] = lst [index]

  • 不要为了将一个赋值表达式与另一个赋值表达式对齐而使用超过一个空格

    Yes:

    x = 1
    y = 2
    long_variable = 3

No:

x             = 1
y             = 2
long_variable = 3

其他建议

  • 总是在二值操作符两侧添加一个空格：=， +=， -=， ==， <, >, !=, <>, <=, >=, in, not in, is, is nota, and, or, not

  • 如果有不同优先级的操作符一起使用，在低优先级的操作符两边加入空格.但是不要使用超过一个空格，并且保证两边空格数目是一致的

    Yes:

    i = i + 1
    submitted += 1
    x = x2 - 1
    hypot2 = x    x + yy
    c = (a+b)     (a-b)

No:

i=i+1
submitted +=1
x = x * 2 - 1
hypot2 = x * x + y * y
c = (a + b) * (a - b)

  • 当=被用于指定关键参数或默认值的时候，不要使用等号

    Yes:

    def complex(real, imag=0.0):

    return magic(r=real, i=imag)

No:

def complex(real, imag = 0.0):
    return magic(r = real, i = imag)
  • 不要在注解函数定义中的=两侧使用空格.并且在:后使用一个空格,在->两侧各添加一个空格.

Yes:

def munge(input: AnyStr):
def munge(sep: AnyStr = None):
def munge() -> AnyStr:
def munge(input: AnyStr, sep: AnyStr = None, limit=1000):

No:

def munge(input: AnyStr=None):
def munge(input:AnyStr):
def munge(input: AnyStr)->PosInt:

  • 不鼓励在同一行中写多个表达式

    Yes:

    if foo == ‘blah’:

    do_blah_thing()

    do_one()
    do_two()
    do_three()

Rather not:

if foo == 'blah': do_blah_thing()
do_one(); do_two(); do_three()

  • 有时if/for/while只有很小的body时可以放在一行，但当其有多种可能表达式时，不要这样做.

    Rather not:

    if foo == ‘blah’: do_blah_thing()
    for x in lst: total += x
    while t < 10: t = delay()

Definitely not:

if foo == 'blah': do_blah_thing()
else: do_non_blah_thing()

try: something()
finally: cleanup()

do_one(); do_two(); do_three(long, argument,
                                                  list, like, this)

if foo == 'blah': one(); two(); three()

注释

注释与代码矛盾比没有注释更糟糕.当代码发生改变时应该优先更新注释.

注释应该完整.如果注释是短语或者句子,首单词应该是大写的，除非是标识符使用小写（never alter the case of identifiers!）

如果注释很短，句尾的句号可以省略.块注释一般由多个完整的句子组成，每句话都应该以句号结尾.

在句子结尾的句号后添加两个空格.

对于来自其他非英语国家的程序员：请使用英语写注释，除非你确定这段代码不会被其他不懂你自己语言的程序员阅读.

块注释

块注释一般针对一些（或者全部）紧跟在后面的代码而写.注释的每一行以#加一个空格开头.

块注释内的段用一个#隔开

内联注释

有节制的使用内联注释.

内联注释注释在同一行.内联注释应该以#加一个空格开头，并且与声明隔开至少两个空格.

如果注释所表述的内容是显而易见的，那么就不需要添加.不要这样做：

x = x + 1                                   # Increment x

但是有时候，这样是有用的：

x = x + 1                                   # Compensate for border

文档字符串

PEP257是编写文档的标准.

  • 为所有public的modules， functions， classes以及methods编写文档字符串.对于非public的metnods不必编写文档字符串，但是应该添加说明的注释.这个注释应该在def行之后.

  • PEP 257 定义了一个很好地文档字符串标准.注意最重要的,用于标记多行文档字符串结束的” “ ”应该自己单独占一行

    “””Return a foobang

    Optional plotz says to frobnicate the bizbaz first.
    “””

  • 对于单行文档字符串，结束处的“ ” “在同一行
